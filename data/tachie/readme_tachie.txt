-------------------------------------------------------------

立ち絵をGu化(ハードウェアー描画)する予定があります。

ハードウェアー描画で描く場合、pspのハードウェアーの制限で、
横が 128 ドットとか 256 ドットとか 512 ドットとかの必要があります。
(横が 2^n ドット以外は出来ない) 横の最大は 512 ドットです。
(横が 2^n ドットで可能なもの：1, 2, 4, 8, 16, 32, 64, 128, 256, 512 のみ、これ以外は出来ない)

縦方向にはこのような制限がありません。(でも縦の最大 512 ドット)

現状の背景などの横480素材は横512ドットに内部で転送して使用しています。
例えば 横257x縦257素材があった場合、 横512x縦512ドット、メモリを使用します。

現状の横150ドットx縦 200ドット立ち絵は、 横256x縦256ドット、メモリを使用しています。
(SDLを使っても自動的にそうなる)

現状の立ち絵は、1枚あたり 横 150 ドット、縦 200 ドットです。

現在２種類の方法を検討中です。

■ ＴｙｐｅＡ(現状重視、共有方式)立ち絵 6枚(150 x 200)
	512x512 ドット(512x402 ドット)のテクスチャ 1枚に、立ち絵 6枚(150 x 200)をまとめる。
	512x512 ドットのテクスチャのうち(453x402)ドットを使用する。
	453 == 150+1 +150+1 + 150+1 	(1ドット開ける必要がある。150ドットの画像を使う場合151ドット必要)
	402 == 200+1 +200+1 			(1ドット開ける必要がある。200ドットの画像を使う場合201ドット必要)

	512x512 ドットのテクスチャ 1枚に、同じサイズの立ち絵 6枚 をまとめる場合、
	1枚あたりの最大サイズは 169x255 ドットです。
	512 >=  510 == 169+1 +169+1 + 169+1 	(1ドット開ける必要がある。169ドットの画像を使う場合170ドット必要)
	512 >=  512 == 255+1 +255+1 			(1ドット開ける必要がある。255ドットの画像を使う場合256ドット必要)

	この方法では 512x512 ドットの画像2枚分が必要です。
	もしVRAMに置かないと速度が間に合わなかった場合に、
	32bit色をあきらめるかVRAMに置くのをあきらめるかどちらか選択が必要ですが、
	速度はたぶん大丈夫な気もします。(?)

■ ＴｙｐｅＢ(妖々夢風、分割方式)立ち絵 は2の倍数 枚(2枚, 4枚, 6枚, ...)
	256x256 ドットのテクスチャ 1枚に、立ち絵 2枚(127 x 255)をまとめる。
	(1ドット開ける必要がある。127ドットの画像を使う場合128ドット必要)
	(1ドット開ける必要がある。255ドットの画像を使う場合256ドット必要)
	永夜紗以降のスペカのような大きな絵の場合は、特別に
	256x256 ドットのテクスチャ 1枚に、立ち絵 1枚(255 x 255)を使う。

	127x255ではレミリア、チルノ等はどうしても狭い感じになる。
	かといって255x255では速度がどうなるやら。
